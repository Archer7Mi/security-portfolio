# Splunk

This folder is for Splunk-focused detection engineering artefacts.

## What I’m putting here
- SPL searches (detections + hunts)
- tuning notes (what caused noise, what reduced false positives)
- dashboard placeholders (screenshots or exports)

## What it demonstrates
- SPL ability (search design, time windows, aggregations)
- detection engineering mindset (signal vs noise)
- writing like a SOC analyst (clear assumptions + next steps)

## How it ties to SOC / governance / risk
- SOC: detections that can be turned into alerts
- Governance: documented logic and reviewable artefacts
- Risk: focusing on higher-impact behaviors (credential abuse, lateral movement)
