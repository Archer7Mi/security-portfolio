# Case Studies

This folder is where I write up **what happened, what I checked, and what I learned**.

I’m keeping these case studies **offline-first** (Markdown + screenshots/log snippets) so they’re easy to review.

## What I’m adding here
- Incident response writeups (phishing, malware, ransomware, identity)
- Detection writeups (how I built/tuned something in KQL/SPL)
- Threat-hunt notes (hypothesis → queries → findings)
- Evidence placeholders (screenshots/logs) that I can fill later

## What it demonstrates
- SOC-style investigation thinking (triage → scope → next steps)
- Clear reporting and evidence handling
- Detection engineering (and how I reduce noise)
- Basic MITRE ATT&CK mapping (when it helps)

## Start here
- Template: [CASE_STUDY_TEMPLATE.md](CASE_STUDY_TEMPLATE.md)

## Naming suggestion
- `YYYY-MM-DD_short-title.md`
- Put images/log exports in an `assets/` folder next to the case.
